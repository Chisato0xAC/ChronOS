## 需求

1. 爬虫只保留视频时长和观看时间。
2. 脚本不包含“保存到文件/数据库”的功能（只负责爬取与计算）。
3. 以 4:00 为界限，只爬取前一天的记录。比如今天 3.11，爬取 3.10 4:00 到 3.11 3:59 的记录。
4. 以第三条为基础。如果程序正在运行，则在到达指定时间时爬取；如果到达指定时间时程序没有运行，在开始运行时检测今天是否爬取，没有则爬取。
5. 所有视频时长叠加，除以 60，取整，作为 DP 变更值。
6. 以第五条为基础，预留扣除 DP 的机制。不用全部做完，因为 DP 的变更记录功能还没做。

## 已完成功能

- 爬虫脚本位于 `src/crawler/bilibili.py`，只使用两类信息：视频时长 `duration` 与观看时间 `view_at`。
- 账号信息从 `data/bilibili_auth.json` 读取（脚本会在文件不存在时自动创建模板）。
- 如果 `data/bilibili_auth.json` 的 `sessdata` 未填写，爬虫会输出一句提示并退出（避免产生大量报错日志）。
- 以北京时间 4:00 为分界，计算“最近一个完整日”的爬取窗口，并只爬取窗口内的记录。
- 启动时会检查 `data/crawler_state.json` 的爬虫状态，若今天窗口未爬取则补爬；持续运行时到每天 4:00 自动爬取。
- 把窗口内所有视频时长相加，除以 60 取整，得到 DP 变更值；本脚本默认输出为扣除（负数）。
- 爬取结果与 DP 变更值写入 `data/crawler_state.json`（用于判断“今天是否已爬取”，并为后续 DP 变更记录预留对接点）。
- `data/crawler_state.json` 会额外记录本次窗口的总分钟数（`last_total_minutes`），方便核对计算结果。
- 预留“扣除 DP”的对接点：爬虫会把本次窗口的扣除值写成 `data/crawler_state.json` 的 `pending_dp_*` 字段（状态为 pending），等待未来 DP 变更记录系统接入。

## 文件职责（约定）

- `data/state.json`：只保存当前状态数据（例如 dp、gp），不保存爬虫运行状态。
- `data/crawler_state.json`：只保存爬虫运行状态（例如今天窗口是否已爬）。
