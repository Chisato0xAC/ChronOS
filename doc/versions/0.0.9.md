## [0.0.9] - 2026-03-25

### Added

- 新增“GitHub 空提交”扩展任务：按配置每 X 天对指定仓库执行一次空提交并 push。
- 新增项目文件结构说明文档 `doc/Project_Structure.md`，用于快速理解目录职责。
- 统一 Python 缓存目录到项目根目录 `__pycache__`。
- 自动重启器也统一 Python 缓存目录到项目根目录 `__pycache__`。
- 新增通用 DP 变更模块，并由进程监控在结束事件时按 Cycle Run Cost Rule 结算 DP。
- /api/service-status 日志增加节流，避免频繁轮询刷屏。
- 修复进程监控对子模块导入失败导致的子进程反复重启。
- 进程监控先设置模块搜索路径，再导入 DP 模块，避免导入失败刷日志。
- 自动重启器增加“重启风暴保护”，短时间内连续失败会自动退出，避免刷屏。
- 进程监控新增每分钟预估扣除 DP 输出（只写预估文件，不修改 DP）。
- 进程监控新增按周期统计的第 N 次倍率结算（周期按 04:00 到次日 04:00）。
- 新增通用状态管理器 `src/state/state_manager.py`，统一承载状态变更与周期结算能力。
- 状态能力拆分为 `state_store`（状态读写）与 `state_rules`（周期和结算规则），降低耦合。
- 主调度器增加子进程重启风暴保护：子进程短时间连续失败会暂停自动重启，避免刷屏。

### Fixed

- 日志改为按天归档：写入 logs/server.YYYY-MM-DD.log。
