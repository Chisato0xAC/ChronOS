## [0.0.8] - YYYY-MM-DD

### Added

1. 前端页面新增“服务状态”显示（Connected / Disconnected）。
2. 新增 `tools/reload_watch.json`，把自动重启监听规则从代码里提取到配置文件。
3. 新增 `doc/concepts/DP_Cycle_Run_Cost.md`，记录“周期内运行消耗规则（草案）”，明确同周期第 N 次运行按 N 倍消耗，并注明沿用本地已有凌晨 4 点结算设置。
4. 新增后端计算接口 `/api/calc-cycle-run-cost`（纯计算不落盘），用于按“第 N 次运行倍率=N”规则计算周期内 DP 消耗，并预留 CHAOS 触发占位结果。
5. 新增独立进程监控程序 `tools/process_watch.py`（不放在 server 内），按 `data/process_watch.json` 里的 `target_process_name` 每秒轮询进程并累计运行秒数。
6. `server.py` 接入“主调度器托管子进程”框架：启动时拉起 `tools/process_watch.py` 作为子进程，退出时统一回收，并预留可扩展的子进程清单与自动拉起机制。
7. 新增 `tools/managed_children.json`，把主调度器子进程清单从代码抽到配置文件；后续新增子进程可直接改配置，不需要改 `server.py`。
8. 新增 `doc/concepts/ManagedChildren.md`，用简单说明文档固定“子进程托管配置格式”和“新增子进程的最小步骤”。
9. `tools/process_watch.py` 改为事件驱动（WMI）：支持按 `watch_list` 监控多个进程的启动/结束，累计每个进程运行秒数，并导出 `data/process_watch_export.csv`。
10. 新增 `tools/test_process_watch.py` 测试用例：验证“启动 + 结束”事件后累计秒数正确、事件文件写入成功、CSV 导出成功。
11. 进程监控完成“规则/数据分离”：规则迁移到 `tools/process_watch_rules.json`（仅 `watch_list`），运行结果保留在 `data/process_watch.json`（仅 `summary/updated_*`）。
12. 统一 JSON 职责边界：将 B 站扣除规则改为 `tools/bilibili_rule.json`（规则），`data/bilibili_rule.json` 仅作旧版本兼容迁移来源，不再作为主读取路径。
13. 将“进程监控”提升为核心模块：文件从 `tools/` 迁移到 `src/monitor/`，与 `src/crawler/` 同层；主调度器改为启动 `src/monitor/process_watch.py`。
14. 进程监控测试文件与规则文件同步迁移到 `src/monitor/`：`test_process_watch.py` 与 `process_watch_rules.json`。
15. 新增用户手册 `doc/rules/ProcessWatch_User_Guide.md`，只说明“进程监控名单”在 `config/process_watch_rules.json` 设置。

### Fixed

1. 对 data/* 的修改会引起非预期的 server 重启。
2. 当后端因代码改动重启时，前端页面不会自动刷新。
3. 自动重启器不再因为 .html 文件改动而重启后端（.md 本来就不在监听列表中）。
4. 自动重启器每轮会重读监听配置，后续改 `reload_watch.json` 可热生效。
5. 前端“服务状态”改为基于 SSE 连接状态，不再每 5 秒轮询 `/api/service-status`。
6. 后端静态文件读取增加安全路径校验：禁止通过 `..` 越界读取项目目录外文件（例如 `/data/../server.py` 这类路径）。
7. 服务端对客户端中途断开连接（WinError 10053）降级处理，不再刷出整段 Traceback，只记录一行简洁日志。
8. 在“修改 DP”的表达式输入框里，按 Enter 键会直接触发“应用表达式”，不需要再手动点按钮。
9. 执行通知测试时，如果通知模块未加载，改为控制台提示，不再使用浏览器弹窗。
10. “应用表达式”成功后会自动清空 DP 输入框，方便继续输入下一条表达式。
