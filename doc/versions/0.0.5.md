## 需求

1. 做一个自定义扩展功能：用户可以根据自己的需求自行编写自定义规则。
    - 示例规则：当项目发起一次提交时，+100DP；如果这次是一次版本发布（打 tag：vX.Y.Z），则额外 +900DP（合计 +1000DP）。

## 已完成功能

1. 预留扩展事件入口：新增 `POST /api/trigger-event`，用于接收外部事件（第一步：只打日志，不修改 DP），为后续规则系统铺路。
2. 新增规则目录与示例规则：增加 `extensions/rules/`，并提供提交 +100DP、版本 tag +900DP 的示例规则 JSON（本步只创建文件，暂未接入执行）。
3. 事件规则匹配（不改 DP）：当收到 `POST /api/trigger-event` 时，服务端会读取 `extensions/rules/*.json` 并打印命中的规则 id（第二步：只匹配+日志，不修改 DP）。
4. 事件去重 + 应用规则：当收到 `POST /api/trigger-event` 时，服务端会先检查 `data/state_history.jsonl` 中是否已存在同一 `event_id` 的 `rule_apply` 记录；未处理过则把命中的规则 `dp_delta` 累加应用到 DP，并写入一条 `rule_apply` history（带 `event_id` 用于去重）。
5. git 提交自动触发（安装脚本）：新增 `tools/install-githooks.bat`，用于安装 `.git/hooks/post-commit`；安装后每次 `git commit` 会自动向 ChronOS 发送 `git_commit` 事件。
6. git tag 自动触发：增加 `.git/hooks/reference-transaction`（通过安装脚本安装）；当创建 `refs/tags/*` 时自动发送 `git_tag` 事件。
7. hooks 自动安装：ChronOS 启动时会自动把 `tools/githooks/` 里的 hook 复制到 `.git/hooks/`（失败会忽略，不影响启动）。
