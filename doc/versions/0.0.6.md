## 已完成功能

1. 已按需求修改 `bilibili.py`：按可配置阈值与权重计算扣除分钟数（短视频/长视频/其它分组分别加权后再除以 60 取整）。
2. 已将换日时间抽离为全局配置：新增 `chronos_config.py`，并让 `server.py`、`src/crawler/bilibili.py` 统一读取。
3. 已支持前端显示历史记录：新增 `GET /api/state-history`，并在页面增加“历史记录”区域；当状态修改/撤销后会自动刷新显示。
   - 历史记录展示会自动隐藏“撤销(undo)记录本身”，并隐藏“已经被撤销的那条记录”。
4. 启动行为优化：服务启动时不会无条件启动爬虫；会先检查 `data/crawler_state.json` 的 `last_trigger_ts`，只有满足“需要爬取”条件时才会启动爬虫脚本。
5. 日志可读性：服务端与爬虫输出都增加了时间戳，方便排查“什么时候发生了什么”。

6. 窗口体验：调整 `ChronOS.vbs` 启动的应用窗口大小，方便展示“历史记录”区域。
