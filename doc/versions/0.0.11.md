## 0.0.11 - 2026-03-31

### Added

1. 通知功能新增“延迟秒数”输入框：可在页面里指定多少秒后发送系统通知，不再固定等待 5 秒。
2. 通知功能改为“时:分:秒”三段输入（小时、分钟、秒），并新增“将在 HH:MM:SS 通知”的到点时间预览。
3. 通知功能新增“待触发通知任务”列表：发布任务后先入列表，到点后再弹出通知。
4. 通知任务改为后端落盘到 `data/notify_tasks.json`，刷新前端后会重新加载未到点任务，不再丢失。

### Fixed

1. 通知按钮文案与行为同步：改为按输入的秒数发送，并在通知正文里显示实际延迟秒数，避免“按钮写可配置但实际固定 5 秒”的误解。
2. 通知按钮文案更新为“按上面的时分秒发送通知”，与实际交互保持一致。
3. 通知“时”输入框补充 `max=99`，与“分/秒”输入框保持接近的显示宽度，避免“时”框明显更宽。
4. 预计通知时间与任务剩余时间改为每秒刷新，修复“时间在走但预览静止”的问题。
