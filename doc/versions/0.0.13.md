## 0.0.13 - 2026-04-03

### Added

1. 主页新增周视图区域：左侧显示本周 7 天日期、`00:00` 到 `23:00` 的时间刻度和对应周网格，并加入“当前时间横条”，方便直接在首页查看一周时间分布和今天当前时间位置。
2. 周视图接入 `process_watch` 进程会话：后端提供本周会话接口，前端会把每条进程运行时段画成色块，直接显示“哪一天、几点到几点在运行什么进程”。
3. 设置面板新增周视图色块颜色选择器，选择结果会保存到 `data/ui_settings.json`，刷新页面后仍会保持你设定的颜色。
4. 首页顶部新增悬浮信息横幅：平时几乎透明，鼠标移上去会显现，点击后可展开信息列表，集中查看服务状态、DP/GP、待触发通知和日报摘要。
5. 首页“修改 DP”区域新增“打卡 +1 DP”按钮：只有每个周期的 `9:00-9:59` 可以点击，并且同一周期内只能领取一次；成功后会把当前 DP 增加 `1`，并写入历史记录。

### Fixed

1. 修复改完代码后页面有时不自动刷新的问题：前端重连成功后，只要页面处于可见状态就会自动整页刷新；如果当时不适合立即刷新，也会在短时间内继续补尝试，减少“明明改了但页面没更新”的情况。
2. 修复周视图中色块与时间网格贴合不准的问题：色块位置改为基于真实格子坐标计算，显示会更准确地对齐到对应日期和时间。
3. 修复周视图色块悬停提示异常的问题：色块恢复正常接收鼠标事件，并修正跨天会话、短间隔合并会话的提示时间显示，避免悬停信息缺失或显示到错误时段。
4. 修复主日志偶尔串行挤在一起的问题：`server.py` 的统一日志输出改为先拿同一把锁再写入，减少多个线程同时打印时把两条日志拼到一行的情况。
5. 继续补修 HTTP 访问日志串行问题：`SaveDpHandler.log_message()` 也改为走同一把日志锁，避免 `GET ... 200 -[下一条日志]` 这种访问日志首尾直接粘连。

### Changed

1. 同步托管子进程规则文档到当前实现：`doc/rules/ManagedChildren.md` 中的配置路径更正为 `config/managed_children.json`，`process_watch` 示例脚本路径更正为 `src/monitor/process_watch.py`。
2. 任务栏补上“商店”占位按钮，并放到 7 个按钮的正中间；它的宽度、间距和点击反馈与“状态”“物品”保持一致，当前统一显示“开发中”。
3. 周视图布局调整为“整页不滚、卡片内部滚动”：同时把“修改 DP”卡片移动到右侧栏，去掉周视图标题和副标题，让时间网格在当前窗口内保持更易读的尺寸。
4. 周视图中的进程色块显示方式进一步简化和优化：跨天会话会按天拆分绘制，相同进程且间隔不超过 7 分钟的短会话会先合并，再以更连贯的色块显示；色块上不再直接写文字，详细信息改为鼠标悬停查看。
5. 打卡奖励值改为配置项：新增 `chronos_config.py` 中的 `CHECKIN_REWARD_DP`，以后要调整打卡奖励时，只需要改这一处，前端按钮文字和后端实际发放值会自动保持一致。
