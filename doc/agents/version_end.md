1. 读取 `doc/versions/CURRENT.md`，确认当前开发版本号。  
2. 检查当前版本代码。
3. 打开对应版本文档 `doc/versions/<版本号>.md`，整理版本文档内容：
   - 只保留最终能力块，不保留重复描述。
   - 文案只写最终效果，不写中间修改过程。
   - 删除没有内容的标题。
4. 在版本文档第一行写上该次最终提交对应的日期，格式保持与现有版本文档一致。
5. 查看 Git 状态，确认将被提交的文件。
6. 执行提交。
7. 使用版本号创建 Git tag，格式为 `v<版本号>`。
8. 推送提交与 tag 到远程。
9. 最后向用户汇报：
   - 当前版本号
   - 提交哈希
   - tag 名称
   - 是否已成功推送
   - 如果失败，明确卡在哪一步
