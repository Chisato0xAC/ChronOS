# 🤖 AI 智能体操作手册 (AGENTS.md)

你好，AI 助手！你是这个项目的执行者。为了确保非编程背景的使用者能完全理解并掌握进度，请严格遵守以下指令：

## 1. 核心原则：单步执行与极简解释

你可以在不经过使用者批准的情况下执行操作，但必须严格遵守以下限制：

* **严禁多任务并行**：一次只实现一个微小的功能（例如：只创建一个按钮，或者只写一个加法逻辑）。禁止在一个步骤中完成多个功能。
* **长远规划，路径固定**：在创建文件前，请从长计议，力求一步到位，请预判项目的最终规模，如果需要文件夹分类，请从第一步就建立好，最大程度避免在后续步骤中移动已有文件的位置。
* **透明化操作**：在执行每一步操作的同时，必须用最直白的、非专业的语言向使用者说明你正在做什么。
* **最大限度易读性**：如果使用者看不懂你的代码，那这就是你的失败。
* **说中文**。

---

## 2. 代码编写标准 (极简主义)

使用者并未学习复杂的编程。为了让他们一眼就能看懂你在做什么，你必须遵守以下指令：

* **禁止使用美化 (CSS)**：不要添加任何样式代码，只保留最原始的文字和功能。
* **禁止使用复杂框架**：严禁引入 Node.js, React, Vue 等对于无基础使用者来说学习难度较高、文件结构复杂的框架。
* **必须写注释**：在代码中使用 // (JavaScript) 或 # (Python) 用中文解释代码在做什么。不需要每一行代码都注释，注意逻辑块的易读性即可。
* **注意规范**：注意遵守代码和文档编写规范，比如 markdownlint。
* **注意操作系统**：该项目在 Windows 系统上开发。请不要使用在 Linux 系统和 MacOS 系统上的指令和规定，如 chmod 和 LF 换行。
* **禁止 Local Storage 持久化**：不要使用 Local Storage 保存核心数据，核心数据必须写入项目内文件。

---

## 3. 开发记录规范

* **固定记录位置**：每完成一个功能，都要更新 `doc/versions/` 下的当前版本文档。
* **版本历史保留**：每个版本一个独立文件，后续版本新增文件，不覆盖旧版本记录。
* **版本文档唯一**：同一个版本只保留一份文档，禁止在多个路径放重复版本文件。
* **BAT 文件规范**：Windows 的 `.bat` 文件必须使用 `CRLF` 换行。

---

## 4. 沟通与执行示例

当你开始一个任务（例如：做一个网页计算器）时，你的操作流程应该是：

> **AI:** “我现在开始为你制作计算器的**第一个部分：数字显示框**。
> 我在 `index.html` 中添加了一个文本框。它目前什么都不能做，只是用来显示数字的。
> [执行代码操作...]
> **下一步**：我将为你添加一个‘加1’的按钮。”

---

## 5. 代码风格参考 (使用者可见)

你的代码必须长成这样（极其简单）：

```html
<div id="result">0</div>

<button onclick="add()">加 1</button>

<script>
  // 这是加法功能的逻辑
  function add() {
    // 第一步：找到那个显示结果的标签
    let box = document.getElementById('result');
    // 第二步：取出里面的数字并加 1
    let num = parseInt(box.innerText) + 1;
    // 第三步：把新数字放回标签里
    box.innerText = num;
  }
</script>

```

---

## 6. 违规后果

如果你的代码中出现了使用者无法理解的复杂逻辑、一次性实现了多个功能、引入了任何框架/样式库，或者或者擅自更改了文件存放路径，使用者有权要求你重置所有进度并重新开始。
